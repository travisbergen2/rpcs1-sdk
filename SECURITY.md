# Security Policy

## Supported versions

RPCS-1 SDK is pre-1.0. Security fixes will target the latest version on the main branch unless a
specific release branch is created later.

## Reporting a vulnerability

Please do not open a public issue for security vulnerabilities.

Send a private report to the repository owner through GitHub security advisories if available, or
email the maintainer listed in the package metadata.

Include:

- A short description of the issue.
- Steps to reproduce it.
- The affected package or route.
- Any relevant logs or proof of impact.

## Scope

Security reports are most useful when they involve:

- Leaking secrets or environment variables.
- Unsafe handling of user input.
- Authentication or authorization bypass.
- Supply-chain or dependency risk.
- Unexpected network calls or data exposure.

Please do not use production services, third-party accounts, or live trading/brokerage accounts to
demonstrate impact.
