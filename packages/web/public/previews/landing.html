Open protocol · Receiver-side communication

AI that adapts to your cognitive style — not the other way around.

RPCS-1 is an open protocol and SDK for compiling a human communication profile into machine-usable rendering directives. It removes prompt friction and executive-function bottlenecks from AI collaboration — deterministically, not by hoping the model infers you.

The problem with prompting a mind-reader

System prompts describe you in prose and hope. RPCS-1 replaces the hope with parameters.

Standard promptingThe RPCS-1 protocol
Static instructionsLong system prompts get skimmed, diluted, or overridden as context grows. Structured receiver profilesPreferences compile into explicit, versioned rendering directives that survive context growth.
Implicit adaptationYou rely on the model to infer your communication parameters from conversation statistics. RPCS-1's design premise — stated as a testable prediction, not a finding — is that this inference is unreliable and should be externalized. External parameter-settingAbstraction level, response pacing, and activation energy are set as explicit knobs, outside the model's discretion.
Cognitive frictionExploratory probes get read as factual claims; executive-function load gets read as disinterest. Precision translationClaims are tagged separately from probes; evidence is separated from speculation; intent survives the channel.

Platform

Each feature is labeled with its actual status. That's the house style.

Masking-mirror intake In SDK

A short structured calibration captures self-reported communication preferences, then reconciles them against observed divergence over time — surfacing where your stated style and your worked style differ.

ReceiverProfile JSON In SDK

Clean, versioned JSON schemas containing explicit rendering directives, portable to any AI agent or LLM workflow. No prose. No vibes.

Governed profile updates In SDK

An externalized update rule adjusts detail density, first-step concreteness, and structure from ongoing interaction — with the learning rate itself a controlled parameter, not model whim.

Privacy-tiered data control Roadmap

Tiered partitioning — Core Card, Working Profile, Private Archive — so profiles stay safe across public and enterprise endpoints. Designed; not yet shipped.

The schema, not the sales pitch

An illustrative ReceiverProfile. Directives are deterministic inputs to the translation layer.

{
  "$schema": "https://rpcs1.dev/v1/receiver-profile.json",
  "version": "1.0",
  "profile": {
    // five primitives, continuous [0,100] — never category labels
    "TI": 20,  // Temporal Integration: bottom line first
    "SG": 25,  // Signal Gain: flat and factual
    "FT": 80,  // Filtering Threshold: explicit & literal, no hints
    "UE": 75,  // Update Elasticity: pushback welcome
    "AR": 75   // Ambiguity Resolution: commit, don't interrogate
  },
  "directives": {
    // derived deterministically by @rpcs1/core — profile is source of truth
    "structure": "bluf",
    "warmth": "minimal",
    "explicitness": "explicit_literal",
    "revision": "open_challenge",
    "ambiguity": "commit"
  },
  "meta": { "source": "intake", "generator": "@rpcs1/core" }
}

// matches the ReceiverProfile + deriveRenderingDirectives output shipped in @rpcs1/core · full JSON Schema at /v1/receiver-profile.json

Grounding — at exactly its earned grade

Most AI products claim more evidence than they have. This one is built by someone who keeps a public claim ledger, and the landing page obeys it.

  • DerivedThe receiver model is built on formal observer requirements: control knobs any bounded estimator-actor must set, with optimal-setting laws from standard optimality theory.
  • RegisteredA validation battery (temporal-integration matching, claim/probe dissociation, urgency laws) is pre-registered with pass/fail criteria fixed before data generation. Results — pass or fail — will be published here.
  • OpenWhether in-context AI receivers can self-tune these parameters from conversation statistics alone is an open, testable question. RPCS-1 externalizes the parameters so you don't have to bet on the answer.
